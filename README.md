# HomeMed Cabinet

A premium, privacy-focused medicine inventory manager for households. All data stays on your device — no cloud, no accounts, no tracking.

![HomeMed Cabinet](https://img.shields.io/badge/version-1.0.0-teal)
![License](https://img.shields.io/badge/license-MIT-green)
![Platform](https://img.shields.io/badge/platform-Web%20%7C%20Android-blue)

## Features

### Dashboard
- Total medicines count with animated stat cards
- Expiring soon and expired medicine alerts
- Emergency readiness score with visual indicators
- Low stock warnings
- Quick export button

### Medicine Management
- Add medicines with detailed forms (name, dosage, form, category, expiration, etc.)
- Edit and delete medicines with confirmation dialogs
- Expandable medicine cards with full details
- Image upload for medicine packaging
- Prescription tracking

### Smart Organization
- Automatic sections for expired, expiring soon, emergency, and children medicines
- Instant search across all fields
- Category filters (Adult, Children, Emergency, Chronic)
- Expiration status filters
- Sort by name, expiration, quantity, or category

### Emergency Readiness
- Built-in emergency essentials checklist
- Readiness percentage score
- Missing items identification
- Status levels: Weak, Moderate, Excellent

### Export System
- **PDF**: Professional formatted report with cover page and tables
- **TXT**: AI-friendly structured text format
- **JSON**: Machine-readable structured data
- Every export includes an AI analysis prompt for external AI assistants

### Privacy-First Design
- 100% local storage — no cloud
- No account required
- No tracking or analytics
- No built-in AI analysis
- You control your data

### Mobile Experience
- Mobile-first responsive design
- Sticky bottom navigation
- Floating action button
- Touch-friendly interface
- Native app feel via Capacitor.js

## Tech Stack

- **Frontend**: React + TypeScript + Vite
- **Styling**: Tailwind CSS + Glassmorphism
- **Animations**: Framer Motion
- **PDF Generation**: jsPDF
- **Mobile**: Capacitor.js for Android
- **Storage**: localStorage (local-first)
- **Icons**: Lucide React

## Getting Started

### Prerequisites

- Node.js 20+
- npm or yarn
- Java 17+ (for Android builds)
- Android Studio (for Android development)

### Web Development

```bash
# Clone the repository
git clone https://github.com/yourusername/homemed-cabinet.git
cd homemed-cabinet

# Install dependencies
npm install

# Start development server
npm run dev

# Build for production
npm run build
```

### Android Build

```bash
# Build the web app first
npm run build

# Sync Capacitor with Android
npx cap sync android

# Open in Android Studio
npx cap open android

# Or build APK directly
cd android
./gradlew assembleDebug
```

The debug APK will be at: `android/app/build/outputs/apk/debug/app-debug.apk`

## CI/CD with GitHub Actions

This project includes a GitHub Actions workflow that automatically builds the Android APK on every push to the `main` branch.

### Setting up CI/CD

1. **Push to GitHub**:
   ```bash
   git init
   git add .
   git commit -m "Initial commit"
   git branch -M main
   git remote add origin https://github.com/yourusername/homemed-cabinet.git
   git push -u origin main
   ```

2. **Download the APK**:
   - Go to your repository on GitHub
   - Click on **Actions** tab
   - Select the latest workflow run
   - Scroll down to the **Artifacts** section
   - Download `homemed-debug-apk` or `homemed-release-apk`

3. **Install on your device**:
   ```bash
   # Enable USB debugging on your Android device
   # Connect via USB, then:
   adb install app-debug.apk
   ```

### Workflow Details

The `.github/workflows/android-build.yml` file performs:
1. Checkout of the repository
2. Node.js 20 setup with npm caching
3. Dependency installation (`npm ci`)
4. Vite production build
5. Capacitor Android sync
6. Java 17 setup for Gradle
7. Gradle dependency caching
8. Debug APK build
9. Release APK build (unsigned)
10. Artifact upload for both APKs

## Privacy Policy

HomeMed Cabinet is designed with privacy as the core principle:

- **No Cloud Storage**: All data is stored in your browser's localStorage
- **No Account Required**: Use immediately without registration
- **No Built-in AI**: Analysis is done by external AI assistants you choose
- **No Tracking**: No analytics, cookies, or tracking pixels
- **You Control Your Data**: Export and manage your data as you see fit
